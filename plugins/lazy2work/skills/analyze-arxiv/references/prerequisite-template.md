# Prerequisite Document Template

Follow this format to write a prerequisite knowledge document for understanding the paper.
Each prerequisite item should include only concepts **essential** for understanding the paper.
Exclude overly basic or common knowledge concepts.

---

```markdown
# {Paper Title} - Prerequisites

> **Target Paper**: {Paper Title} ({arXiv ID})
> **Date**: {YYYYMMDD}
> **Difficulty**: {Beginner/Intermediate/Advanced}

---

## Pre-reading Knowledge Checklist

- [ ] Concept 1: {brief description}
- [ ] Concept 2: {brief description}
- [ ] Concept 3: {brief description}
...

---

## 1. {First Core Prerequisite Concept}

### 1.1 Overview
- Definition and core idea
- Why this is needed to understand the paper

### 1.2 Detailed Explanation
- Core principles/mechanisms
- Key equations (if needed)
- Analogies/examples for intuitive understanding

### 1.3 Relation to the Paper
- How this concept is used/extended in the paper
- Which sections of the paper it appears in

---

## 2. {Second Core Prerequisite Concept}

(Same structure as above)

---

## N. {Nth Core Prerequisite Concept}

(Same structure as above)

---

## Concept Relationship Map

Summarize how each prerequisite concept connects to the paper's methodology:

```
Concept A ──→ Concept B ──→ Paper's Core Technique
                ↑
            Concept C
```

## Recommended Study Order

1. {Most fundamental concept} - Estimated time: ~X min
2. {Intermediate concept} - Estimated time: ~X min
3. {Advanced concept} - Estimated time: ~X min

## References

- [Resource 1 Name](URL) - Brief description
- [Resource 2 Name](URL) - Brief description
```
